# 项目结构与界面原则

## 1. 项目目录

项目按三端拆分：

```text
met-mto
├── mto-admin   PC 管理后台
├── mto-server  后端服务
├── mto-app     内部 App
└── docs        设计文档
```

## 2. 目录职责

| 目录 | 职责 |
| --- | --- |
| mto-admin | PC 管理后台，主要给线上运维和管理员使用 |
| mto-server | 后端 API、数据库访问、MinIO、腾讯地图、报告生成 |
| mto-app | 内部 App，主要给现场实施人员使用 |
| docs | 功能需求、技术架构、项目说明 |

## 3. 界面设计原则

前端不追求复杂动效和重型设计系统，重点是简约、清晰、精致、好维护。

原则：

1. 页面数量少，每个页面只解决一个主要问题。
2. 表单字段少，先满足现场记录和报告生成。
3. 按钮明确，避免隐藏操作。
4. 图片上传要方便，预览要清楚。
5. 状态颜色统一，减少花哨装饰。
6. App 优先适合现场手机操作。
7. 管理后台优先适合查询、创建工单、查看报告。

## 4. App 页面范围

第一版 App 只保留必要页面：

1. 登录页。
2. 工单列表。
3. 工单详情。
4. 到矿打卡。
5. 维修记录填写。
6. 图片上传。
7. 巡检清单填写。
8. 报告预览。
9. 我的。

## 5. 管理后台页面范围

第一版后台只保留必要页面：

1. 登录页。
2. 工单管理。
3. 工单详情。
4. 煤矿管理。
5. 巡检清单管理。
6. 用户管理。
7. 报告查看。

## 6. 视觉方向

视觉关键词：

1. 干净。
2. 克制。
3. 信息清楚。
4. 操作明显。
5. 移动端易点按。
6. 后台易检索。

建议：

1. 主色使用稳定的蓝色或青色。
2. 背景使用浅灰，不大面积使用强渐变。
3. 卡片和表单保持轻量，不堆叠复杂阴影。
4. 工单状态使用颜色标签区分。
5. 图片对比区域突出“维修前”和“维修后”。

## 7. 后续实现方式

后续开发时，由我直接按上述原则进行界面实现，不再单独维护复杂原型文档。
