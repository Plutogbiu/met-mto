# v1.0.2 版本更新需求

## 1. 版本信息

| 项目 | 内容 |
| --- | --- |
| 当前版本 | v1.0.1 |
| 目标版本 | v1.0.2 |
| 状态 | 开发中 |
| 创建时间 | 2026-07-13 |

## 2. 迭代目标

本次版本增强工单完成前的现场留痕校验，避免多人派单时只有一人打卡或未上传任何附件就完成工单。

## 3. 本次更新内容

1. 完成工单前必须至少上传 1 张工单附件。
2. 完成工单前必须校验所有指派工程师均已完成现场打卡。
3. 如果存在未打卡工程师，接口返回具体人员姓名。
4. 附件校验暂按工单附件判断，即 `biz_type = work_order` 且 `biz_id = 工单ID`。
5. 区分后台管理端与现场 App 的登录会话有效期：后台 30 分钟，App 7 天。
6. App 登录页支持记住账号和密码，登录会话过期后自动回填，现场人员可一键重新登录。
7. App 工单详情展示维保内容，例如保内免费、保外收费等。
8. App 工单详情统一使用“工单内容”字段名称，并允许现场工程师编辑未完成工单的工单内容。
9. 删除注意事项字段，后台管理端、App、接口模型和数据库结构同步移除。
10. App 设置支持修改当前账号密码，校验旧密码后使用新密码重新登录。

## 4. 后端规则

完成工单接口：

```http
PUT /api/admin/work-orders/{id}/complete
```

完成前置条件：

1. 工单状态不能为已作废。
2. 工单至少存在 1 条正常附件记录：

```text
file_attachment.biz_type = work_order
file_attachment.biz_id = 工单ID
file_attachment.status = 1
```

3. 工单所有指派工程师都必须存在打卡记录：

```text
work_order_record.work_order_id = 工单ID
work_order_record.record_type = process
work_order_record.operator_id = 指派工程师ID
work_order_record.longitude is not null
work_order_record.latitude is not null
```

登录会话规则：

```text
后台管理端（clientType = admin）：30 分钟
现场 App（clientType = app）：7 天
```

JWT、Redis Token 和登录响应中的 `expiresIn` 均使用对应客户端的有效期。

App 登录凭据规则：

1. 用户勾选“记住账号和密码”且登录成功后，将账号和密码保存到本机缓存。
2. 登录会话过期后，登录页自动回填已保存凭据，用户点击登录即可重新获取 Token。
3. 用户取消勾选时立即删除本机已保存凭据。

工单内容规则：

1. `maintenanceContent` 为维保内容类型，App 按保内免费、保外收费、保外免费、保内收费等文案展示。
2. `content` 统一命名为工单内容，后台和 App 使用相同名称。
3. 现场工程师可编辑自己被指派且未完成工单的工单内容，调用接口：

```http
PUT /api/admin/work-orders/{id}/content
```

4. 接口沿用 `work-order:process` 权限，保存后写入“更新工单内容”流程记录。
5. 已完成、已作废工单不允许编辑工单内容。

注意事项下线规则：

1. 移除 `work_order.notice`、`WorkOrder`、`WorkOrderRequest` 和 `WorkOrderResponse` 中的注意事项字段。
2. 后台工单新增、编辑、详情页和 App 工单详情不再展示注意事项。

App 修改密码规则：

1. App 设置页提供“修改密码”入口，需输入旧密码、新密码和确认新密码。
2. 新密码不能与旧密码相同，两次输入的新密码必须一致。
3. 接口校验当前登录用户和旧密码后更新密码：

```http
PUT /api/app/auth/password
```

4. 修改成功后注销当前 Token，App 清除登录状态并返回登录页。
5. 已开启记住账号和密码时，App 自动将本机缓存密码更新为新密码。

## 5. 提示规则

未上传附件时提示：

```text
请至少上传 1 张现场附件后再完成工单
```

有工程师未打卡时提示：

```text
还有工程师未完成现场打卡：张三、李四
```

## 6. 涉及模块

| 模块 | 说明 |
| --- | --- |
| `mto-server` | 调整完成工单前置校验、登录会话有效期、工单内容编辑接口、App 修改密码接口，并移除注意事项字段 |
| `mto-app` | 自动使用登录响应的 `expiresIn`，支持记住账号和密码、维保内容展示、工单内容编辑和修改密码 |
| `mto-admin` | 移除工单新增、编辑和详情中的注意事项 |
| 数据库 | `migration.sql` 删除 `work_order.notice` 字段 |

## 7. 验收标准

1. 未上传附件时，点击完成工单失败并提示上传附件。
2. 只上传附件但存在指派工程师未打卡时，点击完成工单失败并提示未打卡人员。
3. 所有指派工程师均已打卡且至少有 1 张附件时，可以完成工单。
4. 多人派单时，任意一个指派人员未打卡，工单均不能完成。
5. 单人派单时，该人员打卡并上传附件后可以完成工单。
6. 后台管理端登录超过 30 分钟后需重新登录，现场 App 登录有效期为 7 天。
7. App 勾选记住账号和密码并成功登录后，Token 过期回到登录页时账号和密码自动回填；取消勾选后不再回填。
8. App 工单详情展示正确的维保内容中文文案，原“维保内容”描述统一显示为“工单内容”。
9. App 可以编辑未完成工单的工单内容，保存后后台工单详情和流程记录可以看到最新内容及更新记录。
10. 后台和 App 均不再出现注意事项，升级数据库后 `work_order` 表不再包含 `notice` 列。
11. App 修改密码时旧密码错误会提示，密码修改成功后返回登录页并可使用新密码登录。
