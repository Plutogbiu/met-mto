# 运维内部工单系统设计文档

本目录记录 MET MTO 运维内部工单系统的当前产品边界、技术架构和界面原则。

当前项目定位为小团队内部使用，主要解决客户现场工单、现场维保记录、巡检记录、图片留痕和后续报告沉淀问题。后台管理端的基础模型已先行完成，后续重点会转向移动端现场使用能力。

## 文档清单

1. [功能需求分析](./01-功能需求分析.md)
   - 记录当前业务范围、角色、后台模块、工单与附件分类。
2. [技术架构设计](./02-技术架构设计.md)
   - 记录当前技术选型、目录边界、数据库表和接口设计。
3. [项目结构与界面原则](./03-项目结构与界面原则.md)
   - 记录 `mto-admin`、`mto-server`、`mto-app` 的职责和后台页面组织。

## 当前状态

后台基础模型暂时完成，包括：

1. 登录与 JWT/Redis 会话。
2. 客户管理和客户详情二级页面。
3. 人员管理。
4. 设备管理。
5. 工单管理。
6. 工单详情、处理记录、图片资料审查。
7. 后端文件上传和附件分类管理。

下一阶段建议进入移动端能力开发：

1. App 登录和我的工单。
2. 到客户现场打卡。
3. 腾讯地图定位与导航。
4. 现场图片分类上传。
5. 现场工单和巡检工单处理提交。
6. 工单回执、巡检回执和报告生成。
