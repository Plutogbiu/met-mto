# v1.0.2 版本更新需求

## 1. 版本信息

| 项目 | 内容 |
| --- | --- |
| 当前版本 | v1.0.1 |
| 目标版本 | v1.0.2 |
| 状态 | 开发中 |
| 创建时间 | 2026-07-13 |

## 2. 迭代目标

本次版本增强工单完成前的现场留痕校验，避免多人派单时只有一人打卡或未上传任何附件就完成工单。

## 3. 本次更新内容

1. 完成工单前必须至少上传 1 张工单附件。
2. 完成工单前必须校验所有指派工程师均已完成现场打卡。
3. 如果存在未打卡工程师，接口返回具体人员姓名。
4. 附件校验暂按工单附件判断，即 `biz_type = work_order` 且 `biz_id = 工单ID`。
5. 区分后台管理端与现场 App 的登录会话有效期：后台 30 分钟，App 7 天。
6. App 登录页支持记住账号和密码，登录会话过期后自动回填，现场人员可一键重新登录。

## 4. 后端规则

完成工单接口：

```http
PUT /api/admin/work-orders/{id}/complete
```

完成前置条件：

1. 工单状态不能为已作废。
2. 工单至少存在 1 条正常附件记录：

```text
file_attachment.biz_type = work_order
file_attachment.biz_id = 工单ID
file_attachment.status = 1
```

3. 工单所有指派工程师都必须存在打卡记录：

```text
work_order_record.work_order_id = 工单ID
work_order_record.record_type = process
work_order_record.operator_id = 指派工程师ID
work_order_record.longitude is not null
work_order_record.latitude is not null
```

登录会话规则：

```text
后台管理端（clientType = admin）：30 分钟
现场 App（clientType = app）：7 天
```

JWT、Redis Token 和登录响应中的 `expiresIn` 均使用对应客户端的有效期。

App 登录凭据规则：

1. 用户勾选“记住账号和密码”且登录成功后，将账号和密码保存到本机缓存。
2. 登录会话过期后，登录页自动回填已保存凭据，用户点击登录即可重新获取 Token。
3. 用户取消勾选时立即删除本机已保存凭据。

## 5. 提示规则

未上传附件时提示：

```text
请至少上传 1 张现场附件后再完成工单
```

有工程师未打卡时提示：

```text
还有工程师未完成现场打卡：张三、李四
```

## 6. 涉及模块

| 模块 | 说明 |
| --- | --- |
| `mto-server` | 调整完成工单前置校验和登录会话有效期 |
| `mto-app` | 自动使用登录响应的 `expiresIn`，并支持记住账号和密码 |
| `mto-admin` | 无需调整 |
| 数据库 | 无结构变更 |

## 7. 验收标准

1. 未上传附件时，点击完成工单失败并提示上传附件。
2. 只上传附件但存在指派工程师未打卡时，点击完成工单失败并提示未打卡人员。
3. 所有指派工程师均已打卡且至少有 1 张附件时，可以完成工单。
4. 多人派单时，任意一个指派人员未打卡，工单均不能完成。
5. 单人派单时，该人员打卡并上传附件后可以完成工单。
6. 后台管理端登录超过 30 分钟后需重新登录，现场 App 登录有效期为 7 天。
7. App 勾选记住账号和密码并成功登录后，Token 过期回到登录页时账号和密码自动回填；取消勾选后不再回填。
