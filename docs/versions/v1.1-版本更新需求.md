## 2. 本次迭代内容

1. 工单流转记录中的操作人显示真实姓名，不再保存或展示账号名。
2. 历史工单流转记录按 `operator_id` 回填用户真实姓名。
3. 工单新增维保内容字段 `maintenanceContent`。
4. 现场工单和日常巡检工单都需要选择维保内容。
5. 维保内容作为独立字段传输和保存，不再写入工单内容 `content`。
6. 维保内容暂不做后端固定枚举校验，避免后续选项调整时频繁修改后端代码。

## 3. 工单流转记录操作人

### 3.1 调整目标

工单创建、编辑、状态变更、完成工单、现场处理记录等流转记录中，`operatorName` 需要保存用户真实姓名 `realName`。

### 3.2 后端调整

1. 登录 token 增加 `realName`。
2. 鉴权拦截器解析并写入 `currentRealName`。
3. 工单和工单处理记录接口写入流程记录时使用 `currentRealName`。
4. 旧 token 没有 `realName` 时，根据 `currentUserId` 查询用户真实姓名兜底。

### 3.3 历史数据修正

已有数据库升级时执行 `migration.sql` 中 `V20260709__sync_work_order_record_operator_real_name` 版本段：

```sql
update work_order_record r
inner join sys_user u on u.id = r.operator_id
set r.operator_name = u.real_name
where u.real_name is not null
  and u.real_name <> ''
  and (r.operator_name is null or r.operator_name <> u.real_name);
```

## 4. 工单维保内容

### 4.1 当前前端选项

| 显示名称 | 字段值 |
| --- | --- |
| 保内免费 | `warranty_free` |
| 保外收费 | `out_warranty_paid` |
| 保外免费 | `out_warranty_free` |
| 保内收费 | `warranty_paid` |

后续如需调整维保内容选项，优先调整前端选项或改为字典配置，后端保持普通字符串字段接收。

### 4.2 数据库变更

`work_order` 新增字段：

```sql
maintenance_content varchar(500) null comment '维保内容'
```

已有数据库升级时执行 `migration.sql` 中 `V20260709__add_work_order_maintenance_content` 版本段。
